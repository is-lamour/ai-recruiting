# ai-recruiting

Автоматический скрининг резюме с hh.kz через Gemini AI. Расширение Chrome собирает резюме со страницы откликов, бэкенд оценивает каждое по требованиям вакансии (0–100 баллов), результаты отображаются в веб-дашборде.

## Быстрый старт

### 1. Gemini API ключ

Перейдите на <https://aistudio.google.com/app/apikey>, создайте ключ и скопируйте его.

### 2. Запустите бэкенд

**Windows:** дважды кликните `start.bat` — он проверит зависимости, запустит сервер и откроет браузер автоматически.

**Вручную:**

```bash
pip install -r backend/requirements.txt
cd backend
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

Дашборд доступен по адресу: **<http://localhost:8000>**

### 3. Установите расширение Chrome

1. Откройте `chrome://extensions/`
2. Включите **Режим разработчика** (правый верхний угол)
3. Нажмите **Загрузить распакованное**
4. Выберите папку `extension/` из этого проекта
5. Откройте расширение → введите Gemini API ключ при первом запуске

> **Примечание:** `.env` файл не обязателен — API ключ хранится в расширении Chrome и передаётся на бэкенд автоматически.

## Раздача другим пользователям

Для передачи программы нетехническому пользователю (без установки Python):

### 1. Соберите дистрибутив

```bash
build_dist.bat
```

Скрипт автоматически:
- Скачивает Python 3.11 Embedded (официальный интерпретатор от python.org)
- Устанавливает все зависимости внутрь
- Упаковывает всё в `ai-recruiting-dist.zip` (~80–100 МБ)

### 2. Отправьте архив пользователю

Пользователю достаточно:
1. Распаковать `ai-recruiting-dist.zip` в любую папку
2. Дважды кликнуть `start.bat` — сервер запустится, браузер откроется автоматически
3. Установить расширение Chrome из папки `extension/` (один раз)

Подробная инструкция для пользователя — в файле `INSTRUKCIYA.txt` внутри архива.

> **Почему не антивирусит:** используется официальный `python.exe` от python.org, а не скомпилированный бинарник. Все зависимости — обычные `.py` файлы.

## Как пользоваться

### Создание вакансии

В расширении нажмите «+ Добавить вакансию». Вакансию можно создать тремя способами:

- **По ссылке** — вставьте URL вакансии с hh.kz, нажмите «Загрузить» — название и описание подтянутся автоматически
- **Вручную** — введите название и вставьте текст вакансии
- **Summary вручную** — заполните поле Summary самостоятельно, иначе AI сформирует его автоматически в фоне (~5 сек после сохранения)

Summary — это сжатое описание требований (до 700 символов), которое используется при скрининге каждого резюме.

Для пересчёта summary нажмите **↺ Перегенерировать** — поле очистится, после сохранения AI создаст новый вариант.

### Запуск скрининга

1. Выберите вакансию в расширении
2. Откройте страницу откликов на hh.kz (список резюме по нужной вакансии)
3. Нажмите **▶ Запустить скрининг**

Расширение автоматически:
- Обходит все страницы пагинации (до 20 страниц)
- Загружает каждое резюме с авторизацией через сессию hh.kz
- Отправляет текст резюме на бэкенд для оценки AI
- Показывает прогресс в реальном времени

Уже скринированные резюме пропускаются повторно.

Для остановки нажмите **■ Стоп**.

### Просмотр результатов

Нажмите **📊 Дашборд** — откроется `http://localhost:8000` с результатами по выбранной вакансии.

### Массовые действия

После скрининга в расширении появляются кнопки:

- **Выполнить отказы** — расширение нажимает кнопку «Отказать» на hh.kz для всех кандидатов, помеченных на отказ
- **Открыть в Huntflow** — открывает вкладки с резюме подходящих кандидатов, отмечает их как отправленных

## Дашборд

Веб-интерфейс на `http://localhost:8000`. Показывает всех кандидатов по выбранной вакансии.

### Категории кандидатов

| Категория | Баллы | Описание |
|-----------|-------|----------|
| **Подходит** | 80–100 | Рекомендуется к собеседованию |
| **Подумать** | 66–79 | Стоит рассмотреть |
| **Отказ** | 0–65 | Не соответствует требованиям |

### Карточка кандидата

Каждая карточка содержит:
- **Балл** (0–100) — цветной кружок
- **Краткое резюме** — стек и опыт кандидата в одну строку
- **Комментарий AI** — почему такой балл
- **Вопросы по пробелам** — что уточнить на интервью
- **Время скрининга** — дата и время анализа резюме
- **Ссылка на резюме** — открывает профиль на hh.kz

### Фильтрация и сортировка

**Фильтры по категории:** Все / Подходят / Подумать / Отказ

**Сортировка:**
- По баллу ↓ (по умолчанию)
- По баллу ↑
- Сначала новые (по времени скрининга)
- Сначала старые

### Действия с кандидатами

- **→ В Huntflow** — пометить кандидата для отправки в Huntflow
- **✕ Отказать** — пометить кандидата на отказ в HH
- **Отменить** — снять пометку
- **🗑 Удалить** — удалить кандидата из базы

**Массовые действия:**
- **→ В Huntflow (все «Подходят»)** — помечает всех подходящих кандидатов
- **✕ На отказ в HH (все «Отказ»)** — помечает всех кандидатов категории «Отказ»
- **↓ Экспорт в Excel** — скачивает Excel с результатами (балл, категория, комментарий, вопросы, ссылка)

### Управление вакансией

В дашборде можно редактировать вакансию (кнопка ✏️), удалять её вместе со всеми кандидатами, просматривать полное описание и summary.

## Архитектура

Проект состоит из двух независимых частей, которые общаются по HTTP.

```
extension/          — Chrome расширение (Manifest V3)
  popup.html/js/css — интерфейс расширения
  content.js        — инжектируется в hh.kz, собирает резюме
  background.js     — service worker, CORS-прокси к localhost:8000

backend/            — FastAPI сервер
  main.py           — все HTTP маршруты
  ai.py             — интеграция с Gemini API
  database.py       — SQLite, инициализация и миграции
  static/           — дашборд (SPA на чистом HTML/JS/CSS)
    index.html
    app.js
    style.css
```

### Поток данных

```
hh.kz (откликов страница)
  → content.js собирает ссылки на резюме (пагинация до 20 стр.)
  → content.js загружает каждое резюме через fetch (с cookie сессии hh.kz)
  → content.js извлекает текст из DOM
  → background.js (CORS-прокси) → POST /api/screen
  → backend: ai.screen_resume() → Gemini API
  → результат сохраняется в SQLite
  → дашборд http://localhost:8000 показывает результаты
```

### Почему нужен background.js

Контент-скрипты на hh.kz не могут напрямую обращаться к `localhost:8000` из-за CSP. `background.js` выступает прокси: принимает сообщения от `content.js` и выполняет реальные fetch-запросы к бэкенду.

### Статусы кандидата

```
new → to_reject → rejected
new → to_huntflow → huntflow_sent
```

## API

Все эндпоинты бэкенда (`http://localhost:8000`):

### Вакансии

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/vacancies` | Список всех вакансий |
| `POST` | `/api/vacancies` | Создать вакансию |
| `PUT` | `/api/vacancies/{id}` | Обновить вакансию |
| `DELETE` | `/api/vacancies/{id}` | Удалить вакансию и её кандидатов |
| `GET` | `/api/vacancies/{id}/candidates` | Список кандидатов (опц. `?category=`) |
| `GET` | `/api/vacancies/{id}/stats` | Статистика по категориям |
| `GET` | `/api/vacancies/{id}/screened-urls` | URL уже скринированных резюме |
| `GET` | `/api/vacancies/{id}/pending-actions` | Кандидаты на отказ / в Huntflow |
| `POST` | `/api/vacancies/{id}/mark-for-reject` | Пометить всех «Отказ» на отказ |
| `POST` | `/api/vacancies/{id}/mark-for-huntflow` | Пометить всех «Подходят» в Huntflow |
| `GET` | `/api/vacancies/{id}/export` | Скачать Excel |
| `GET` | `/api/parse-hh-vacancy?url=` | Распарсить вакансию по URL hh.kz |

### Кандидаты

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/screen` | Скринировать резюме |
| `PATCH` | `/api/candidates/{id}` | Обновить статус |
| `DELETE` | `/api/candidates/{id}` | Удалить кандидата |

### Тело POST /api/screen

```json
{
  "vacancy_id": 1,
  "name": "Иван Иванов",
  "hh_url": "https://hh.kz/resume/...",
  "resume_text": "текст резюме"
}
```

Ответ:

```json
{
  "id": 42,
  "score": 78,
  "category": "consider",
  "ai_comment": "Опыт подходит, но нет опыта с PostgreSQL",
  "questions": ["Работали ли с PostgreSQL?", "..."],
  "summary": "Python 4г, Django, MySQL, опыт в e-commerce",
  "status": "new",
  "created_at": "2026-06-01T10:23:00"
}
```

Если резюме уже есть в базе — возвращает существующую запись без повторного обращения к AI.

**Заголовок `X-Gemini-Key`** — если передан, используется вместо ключа из `.env`.

## AI и промпты

Модель: **gemini-3.1-flash-lite**

### Промпт скрининга резюме

```
Ты HR. Оцени резюме по требованиям вакансии. Ответь СТРОГО одной строкой JSON без markdown:
{"score":75,"category":"consider","comment":"До 80 символов.","questions":["?","?","?"],"summary":"Стек и опыт кандидата до 100 символов"}
Правила: suitable>=80, consider 66-79, reject<66. summary — ключевые навыки/стек/опыт кандидата.

ТРЕБОВАНИЯ ВАКАНСИИ: {requirements}
РЕЗЮМЕ: {resume_text}
```

Лимиты: `requirements` — первые 700 символов, `resume_text` — первые 3000 символов.

### Промпт генерации summary вакансии

```
Сожми требования вакансии в одну строку до 600 символов.
Включи: ключевые навыки, стек технологий, опыт (лет), ключевые обязанности. Только суть, без воды.

ВАКАНСИЯ: {description}
```

Результат обрезается до 700 символов и сохраняется в поле `requirements` вакансии.

Генерация запускается асинхронно в фоне (`BackgroundTasks`) после создания или обновления вакансии, если поле `requirements` не заполнено вручную.

## База данных

SQLite, файл: `backend/screening.db`

### Таблица `vacancies`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PK | Автоинкремент |
| `title` | TEXT | Название вакансии |
| `description` | TEXT | Полное описание |
| `requirements` | TEXT | Summary для скрининга (AI или вручную) |
| `created_at` | TIMESTAMP | Дата создания |

### Таблица `candidates`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PK | Автоинкремент |
| `vacancy_id` | INTEGER | FK → vacancies.id |
| `name` | TEXT | Имя кандидата |
| `hh_url` | TEXT | Ссылка на резюме |
| `resume_text` | TEXT | Полный текст резюме |
| `score` | INTEGER | Балл 0–100 |
| `category` | TEXT | `suitable` / `consider` / `reject` / `pending` |
| `ai_comment` | TEXT | Комментарий AI |
| `questions` | TEXT | JSON-массив вопросов |
| `summary` | TEXT | Краткое описание стека кандидата |
| `status` | TEXT | `new` / `to_reject` / `to_huntflow` / `rejected` / `huntflow_sent` |
| `created_at` | TIMESTAMP | Дата скрининга |

Миграции для колонок, добавленных после начального релиза, выполняются автоматически при старте через `init_db()`.
