# HH Автоскрининг

Инструмент автоматического скрининга резюме с hh.kz на основе Google Gemini AI. Расширение Chrome собирает резюме со страницы откликов, backend оценивает каждого кандидата по заданным критериям, веб-дашборд показывает результаты с разбивкой по категориям.

## Возможности

- **AI-скрининг** — оценка резюме по шкале 0–100 с категориями «Подходит» / «Подумать» / «Отказ»
- **Метрики вакансии** — настраиваемые критерии оценки с весами; AI автоматически предлагает 5–7 метрик по описанию вакансии, их можно редактировать вручную
- **Score breakdown** — детальный разбор оценки по каждому критерию с обоснованием, плюсы и минусы кандидата
- **Boolean search** — автогенерация широкого / среднего / узкого поисковых запросов для hh.kz с учётом синтаксиса платформы
- **Перескрининг** — повторная оценка уже загруженных резюме (например, после изменения метрик), с прогресс-баром
- **Массовые действия** — автоматические отказы через DOM hh.kz или открытие вкладок для отправки в Huntflow
- **Корзина** — мягкое удаление кандидатов с возможностью восстановления или полного удаления
- **Экспорт в Excel** — форматированный отчёт со всеми полями: балл, категория, плюсы/минусы, breakdown, вопросы, ссылки
- **Парсинг вакансии по URL** — автозаполнение названия и описания из ссылки на hh.kz
- **Пагинация резюме** — обход до 20 страниц откликов, уже скринированные резюме пропускаются

## Быстрый старт

### 1. Gemini API ключ

Перейдите на <https://aistudio.google.com/apikey>, создайте ключ и скопируйте его.

### 2. Запустите бэкенд

**Windows:** дважды кликните `start.bat` — он проверит зависимости, запустит сервер и откроет браузер автоматически.

**Вручную:**

```bash
pip install -r backend/requirements.txt
cd backend
python -m uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

Дашборд доступен по адресу: **<http://localhost:8000>**

### 3. Установите расширение Chrome

1. Откройте `chrome://extensions/`
2. Включите **Режим разработчика** (правый верхний угол)
3. Нажмите **Загрузить распакованное**
4. Выберите папку `extension/` из этого проекта
5. Откройте расширение → введите Gemini API ключ при первом запуске

> `.env` файл не обязателен — API ключ хранится в расширении Chrome и передаётся на бэкенд автоматически через заголовок `X-Gemini-Key`.

### Проверка подключения к Gemini

```bash
python test_gemini.py
python test_screening.py
```

Для передачи программы нетехническому пользователю (без установки Python):

## Как пользоваться

### Создание вакансии

В расширении нажмите «+ Добавить вакансию». Вакансию можно создать тремя способами:

- **По ссылке** — вставьте URL вакансии с hh.kz, нажмите «Загрузить» — название и описание подтянутся автоматически
- **Вручную** — введите название и вставьте текст вакансии
- **Summary вручную** — заполните поле Summary самостоятельно, иначе AI сформирует его автоматически в фоне (~5 сек после сохранения)

Summary — это сжатое описание требований (до 700 символов), которое используется при скрининге каждого резюме.

После создания вакансии в фоне автоматически генерируются **метрики оценки** (5–7 критериев с весами). Их можно просмотреть и отредактировать на вкладке «Метрики» в дашборде.

### Настройка метрик

Метрики — это критерии, по которым AI оценивает резюме. Каждый критерий имеет вес (влияние на итоговый балл):

- Вес 7.0 — ключевое требование вакансии
- Вес 3.0–6.0 — важные навыки
- Вес 1.0–2.5 — желательные / бонусные навыки

Итоговый балл рассчитывается как взвешенное среднее: `sum(score × weight) / sum(weight) × 10`.

Метрики можно сгенерировать заново кнопкой **↺ Перегенерировать метрики**.

### Boolean search

На вкладке «Boolean Search» в дашборде можно сгенерировать три поисковых запроса для hh.kz:

- **Широкий** — максимальный охват, много синонимов
- **Средний** — баланс охвата и точности
- **Узкий** — только обязательные технологии, минимум шума

Запросы строятся по стеку и функционалу вакансии, без названия должности.

### Запуск скрининга

1. Выберите вакансию в расширении
2. Откройте страницу откликов на hh.kz (список резюме по нужной вакансии)
3. Нажмите **▶ Запустить скрининг**

Расширение автоматически:
- Обходит все страницы пагинации (до 20 страниц)
- Загружает каждое резюме с авторизацией через сессию hh.kz
- Отправляет текст резюме на бэкенд для оценки AI
- Показывает прогресс в реальном времени

Уже скринированные резюме пропускаются. Для остановки нажмите **■ Стоп**.

### Перескрининг

Если вы изменили метрики вакансии, можно переоценить всех кандидатов с новыми весами без повторного сбора резюме. В дашборде нажмите **↺ Перескринировать** — прогресс отображается в реальном времени.

### Просмотр результатов

Нажмите **📊 Дашборд** — откроется `http://localhost:8000` с результатами по выбранной вакансии.

### Массовые действия

После скрининга в расширении появляются кнопки:

- **Выполнить отказы** — расширение нажимает кнопку «Отказать» на hh.kz для всех кандидатов, помеченных на отказ
- **Открыть в Huntflow** — открывает вкладки с резюме подходящих кандидатов, отмечает их как отправленных

## Дашборд

Веб-интерфейс на `http://localhost:8000`. Показывает всех кандидатов по выбранной вакансии.

### Категории кандидатов

| Категория | Баллы | Описание |
|-----------|-------|----------|
| **Подходит** | 80–100 | Рекомендуется к собеседованию |
| **Подумать** | 66–79 | Стоит рассмотреть |
| **Отказ** | 0–65 | Не соответствует требованиям |

### Карточка кандидата

Каждая карточка содержит:
- **Балл** (0–100) — цветной кружок
- **Краткое резюме** — стек и опыт кандидата в одну строку
- **Плюсы / Минусы** — конкретные сильные и слабые стороны относительно вакансии
- **Score breakdown** — оценка по каждому критерию с весом и обоснованием
- **Комментарий AI** — итоговый вывод
- **Вопросы по пробелам** — что уточнить на интервью
- **Время скрининга** — дата и время анализа резюме
- **Ссылка на резюме** — открывает профиль на hh.kz

### Фильтрация и сортировка

**Фильтры по категории:** Все / Подходят / Подумать / Отказ

**Сортировка:**
- По баллу ↓ (по умолчанию)
- По баллу ↑
- Сначала новые (по времени скрининга)
- Сначала старые

### Действия с кандидатами

- **→ В Huntflow** — пометить кандидата для отправки в Huntflow
- **✕ Отказать** — пометить кандидата на отказ в HH
- **Отменить** — снять пометку
- **🗑 В корзину** — мягкое удаление (можно восстановить)

### Корзина

Удалённые кандидаты попадают в корзину. Там их можно:
- **Восстановить** — вернуть в основной список
- **Удалить навсегда** — полное удаление из базы
- **Очистить корзину** — удалить всех из корзины для вакансии

### Массовые действия в дашборде

- **→ В Huntflow (все «Подходят»)** — помечает всех подходящих кандидатов
- **✕ На отказ в HH (все «Отказ»)** — помечает всех кандидатов категории «Отказ»
- **↓ Экспорт в Excel** — скачивает форматированный Excel с результатами

### Excel-экспорт

Экспортируемый файл содержит:
- Заголовок вакансии, описание, требования
- Список использованных метрик с весами
- По каждому кандидату: балл, категория, имя, статус, дата, краткое резюме, плюсы, минусы, комментарий AI, оценка по критериям (breakdown), вопросы, ссылка, полный текст резюме
- Цветовая кодировка строк по категории, гиперссылки на резюме

### Управление вакансией

В дашборде можно:
- Редактировать вакансию (кнопка ✏️)
- Настраивать и перегенерировать метрики
- Генерировать Boolean search запросы
- Удалять вакансию вместе со всеми кандидатами

## Архитектура

Проект состоит из двух независимых частей, которые общаются по HTTP.

```
extension/          — Chrome расширение (Manifest V3)
  popup.js          — интерфейс расширения
  content.js        — инжектируется в hh.kz, собирает резюме
  background.js     — service worker, CORS-прокси к localhost:8000
  dashboard_bridge.js

backend/            — FastAPI сервер
  main.py           — все HTTP маршруты
  ai.py             — интеграция с Gemini API
  database.py       — SQLite, инициализация и миграции
  static/           — дашборд (SPA на чистом HTML/JS/CSS)
```

### Поток данных

```
hh.kz (страница откликов)
  → content.js собирает ссылки на резюме (пагинация до 20 стр.)
  → content.js загружает каждое резюме через fetch (с cookie сессии hh.kz)
  → content.js извлекает текст из DOM
  → background.js (CORS-прокси) → POST /api/screen
  → backend: ai.screen_resume() → Gemini API
  → результат сохраняется в SQLite
  → дашборд http://localhost:8000 показывает результаты
```

### Почему нужен background.js

Контент-скрипты на hh.kz не могут напрямую обращаться к `localhost:8000` из-за CSP. `background.js` выступает прокси: принимает сообщения от `content.js` и выполняет реальные fetch-запросы к бэкенду.

### Статусы кандидата

```
new → to_reject → rejected
new → to_huntflow → huntflow_sent
```

## AI и промпты

Модель: **gemini-3.1-flash-lite**

### Скрининг без метрик

Используется, если у вакансии нет настроенных метрик. AI самостоятельно определяет критерии оценки.

Входные данные: `requirements` (до 700 символов) + `resume_text` (до 3000 символов).

Выходные поля: `score`, `category`, `comment`, `summary`, `questions`, `pros`, `cons`, `score_breakdown` (4–6 критериев).

### Скрининг с метриками

Если у вакансии заданы метрики, AI оценивает резюме строго по ним. Вес каждого критерия берётся из метрик без изменений. Итоговый score пересчитывается на стороне бэкенда по формуле взвешенного среднего — ответ AI не используется напрямую.

### Генерация метрик

Промпт формирует список из 5–7 критериев с весами. Максимальный вес — 7.0 (одна метрика), остальные — 3.0–6.0. После генерации бэкенд нормирует веса: если максимум > 7, всё масштабируется пропорционально.

### Сжатие вакансии (summary)

Описание вакансии сжимается до ≤700 символов с акцентом на ключевые навыки, стек, опыт и обязанности. Генерируется асинхронно в фоне после создания вакансии, если поле `requirements` не заполнено вручную.

### Boolean search

По описанию вакансии генерируется три запроса (широкий / средний / узкий) с использованием синтаксиса hh.kz: `OR`, `AND`, `"..."`, `(...)`, `*`, `!`. Название должности в запросы не включается — только стек, навыки и функционал.

## База данных

SQLite, файл: `backend/screening.db`. Миграции применяются автоматически при каждом запуске через `init_db()`.

### Таблица `vacancies`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PK | Автоинкремент |
| `title` | TEXT | Название вакансии |
| `description` | TEXT | Полное описание |
| `requirements` | TEXT | Summary для скрининга (AI или вручную) |
| `metrics` | TEXT | JSON-массив критериев с весами |
| `boolean_search` | TEXT | JSON с тремя поисковыми запросами |
| `created_at` | TIMESTAMP | Дата создания |

### Таблица `candidates`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER PK | Автоинкремент |
| `vacancy_id` | INTEGER | FK → vacancies.id |
| `name` | TEXT | Имя кандидата |
| `hh_url` | TEXT | Ссылка на резюме (уникальна в рамках вакансии) |
| `resume_text` | TEXT | Полный текст резюме |
| `score` | INTEGER | Балл 0–100 |
| `category` | TEXT | `suitable` / `consider` / `reject` / `pending` |
| `ai_comment` | TEXT | Итоговый комментарий AI |
| `summary` | TEXT | Краткое описание стека кандидата |
| `questions` | TEXT | JSON-массив вопросов по пробелам |
| `pros` | TEXT | JSON-массив плюсов |
| `cons` | TEXT | JSON-массив минусов |
| `score_breakdown` | TEXT | JSON-массив оценок по критериям |
| `status` | TEXT | `new` / `to_reject` / `to_huntflow` / `rejected` / `huntflow_sent` |
| `is_trashed` | INTEGER | 0 — активен, 1 — в корзине |
| `created_at` | TIMESTAMP | Дата скрининга |
